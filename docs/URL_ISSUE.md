# Проблема с URL Django админки

## Обнаруженная проблема

При попытке открыть `/admin/core/abik/` парсер получает **фронтенд приложение (Next.js)**, а не Django админку.

### Что видно в HTML:
- Страница 404 "Страница не найдена"
- Это Next.js приложение (не Django)
- URL редиректит на фронтенд

## Возможные причины

1. **Неправильный URL** - возможно, Django админка находится по другому пути
2. **Редирект** - Django админка редиректит на фронтенд
3. **Другой домен** - Django админка может быть на другом поддомене

## Решения

### Вариант 1: Проверить правильный URL

Откройте в браузере (после авторизации в Django админке):
- `https://smashers.bookbot.olegb.dev/admin/` - главная страница админки
- Проверьте, какие разделы там есть
- Найдите правильный путь к абонементам и тренировкам

### Вариант 2: Проверить структуру Django админки

Возможные правильные пути:
- `/admin/core/abik/` - текущий (не работает)
- `/admin/core/membership/` - может быть так
- `/admin/core/memberships/` - может быть так
- `/admin/abik/` - без core
- `/admin/core/abik/list/` - с list

### Вариант 3: Использовать другой домен

Если Django админка на другом домене:
- Обновите `DJANGO_URL` в `.env`
- Например: `https://admin.smashers.bookbot.olegb.dev`

## Следующие шаги

1. **Откройте Django админку вручную** в браузере
2. **Проверьте URL** страницы с абонементами
3. **Скопируйте правильный URL** и обновите парсеры
4. **Проверьте URL** для тренировок (sessions)

## Как найти правильный URL

1. Войдите в Django админку: `https://smashers.bookbot.olegb.dev/admin/`
2. Найдите раздел с абонементами
3. Скопируйте URL из адресной строки браузера
4. Обновите парсер `src/parsers/django/memberships.js`:
   ```javascript
   const page = await navigateToAdminPage('/правильный/путь/');
   ```

## Проверка

После обновления URL запустите снова:
```bash
node scripts/debug-parser.js memberships
```

Должен открыться правильный URL с таблицей Django админки.
